    class Player

    The Player model is designed to hold the playerâ€™s name and birth year. for a single player for a given year. The player can either be imported or entered manually. The import format can be a CSV file with the following columns:

    • playerID - unique id

    • birthYear

    • nameFirst

    • nameLast

    The entire set of player records will be index by player_id, which should be unique.

    The import method handles importing a CSV file of player records. It will overwrite the current set of players with the new ones imported from the file.

    # File app/models/player.rb, line 59
def self.import(file)
  Player.delete_all
  @@skipped_rows = 0
  
  CSV.foreach(file.path, headers: true) do |row|
    if row['playerID'].present?
      player = Player.new
      player.player_id = row['playerID']
      player.birth_year = row['birthYear']
      player.first_name = row['nameFirst']
      player.last_name = row['nameLast'] 
      if !player.save         
        logger.debug("[Player.import] Could not save imported record - error: #{player.errors.full_messages}")
      end
    else  
      @@skipped_rows += 1
    end
  end    
end
